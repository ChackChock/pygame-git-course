# Урок 2: GitHub и Markdown

Добро пожаловать на второй урок! Здесь вы найдёте теорию и практические материалы по работе с удалёнными репозиториями на GitHub и оформлению проектов с помощью языка разметки Markdown.

## Содержание

*   **Теория:** [`Теория.html`](Теория.html)  
    Полное руководство по работе с GitHub: создание аккаунта, связь локального и удалённого репозиториев, команда `git push`, а также основы синтаксиса Markdown для написания `README.md`.
*   **Практика:** Задание по созданию и оформлению файла `README.md` для вашего проекта.
*   **Проверка знаний:** [`Тестирование.html`](Тестирование.html)  
    Интерактивный тест для проверки ваших знаний по GitHub и Markdown.

## Пройденные команды и понятия

На этом уроке вы изучили следующие команды и концепции:
*   `git remote add origin <ссылка>`
*   `git push -u origin main` и `git push`
*   Назначение файла `README.md`
*   Основной синтаксис Markdown: заголовки, списки, выделение, ссылки, блоки кода

Удачи в освоении!
