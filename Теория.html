Основы pygame-ce

1. Что такое pygame-ce?

pygame-ce (Community Edition) — это современная и активно поддерживаемая версия библиотеки Pygame, предназначенная для создания 2D-игр и мультимедийных приложений на языке Python.

Она берёт на себя всю «низкоуровневую» работу: создание окна, обработка ввода с клавиатуры и мыши, отрисовка графики и воспроизведение звука. Это позволяет вам сосредоточиться на самой игровой логике.

2. Игровой цикл — сердце любой игры

Вся игра на pygame-ce строится вокруг бесконечного цикла, который повторяется много раз в секунду (обычно 60 раз). Этот цикл состоит из трёх ключевых этапов:

  1. Обработка событий (Event Handling): Проверка, что произошло: нажата клавиша, кликнул ли пользователь мышью, закрыл ли окно. Реализуется через for event in pygame.event.get().
  2. Обновление состояния игры (Update): Здесь происходит вся логика: персонажи двигаются, проверяются столкновения, генерируются враги.
  3. Отрисовка (Rendering): Всё, что было обновлено, рисуется на экране. Сначала очищается старый кадр, затем отрисовываются объекты, и в конце экран обновляется.

Поняв этот цикл, вы поймёте, как работают все игры на Pygame.

3. Основные компоненты кода

Инициализация и окно

Любая игра начинается с запуска Pygame и создания окна.

import pygame
pygame.init()

# Создание современного окна
window = pygame.Window("Моя игра", (800, 600))
surface = window.get_surface() # Поверхность для рисования

Управление скоростью игры (Clock)

Чтобы игра работала с постоянной скоростью на любом компьютере, используется класс pygame.Clock. Метод clock.tick(FPS) приостанавливает выполнение цикла, чтобы он не выполнялся чаще указанного количества кадров в секунду (FPS).

clock = pygame.time.Clock()
FPS = 60

# В конце игрового цикла
clock.tick(FPS)

Поверхности и прямоугольники

Surface — это область пикселей, на которой можно рисовать (например, спрайт или всё окно).
Rect — это прямоугольник, который хранит координаты и размеры объекта. Он используется для позиционирования и проверки столкновений.

# Создание простого спрайта
player_image = pygame.Surface((50, 50))
player_rect = player_image.get_rect()
player_rect.center = (400, 300)

# Отрисовка
surface.blit(player_image, player_rect)

Векторы для движения

Класс pygame.Vector2 упрощает расчёты направления и скорости движения объектов.

# Вектор от игрока к позиции мыши
vector = pygame.Vector2(mouse_x - player_x, mouse_y - player_y)
# Нормализация для получения единичного вектора направления
direction = vector.normalize()

4. Важные принципы

  • Порядок отрисовки важен: Объекты, нарисованные позже, будут перекрывать объекты, нарисованные раньше.
  • Очистка экрана: Перед каждой новой отрисовкой необходимо очищать экран с помощью surface.fill(color), иначе кадры будут накладываться друг на друга.
  • Обновление экрана: Все изменения на поверхности становятся видны только после вызова window.flip().

Соблюдение этих простых правил поможет вам избежать самых распространённых ошибок при создании игр.
