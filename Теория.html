Основы Git

1. Что такое Git?

Git — это распределённая система контроля версий, предназначенная для отслеживания изменений в файлах и координации работы над ними между несколькими разработчиками.

Система была создана Линусом Торвальдсом в 2005 году для управления разработкой ядра операционной системы Linux. На сегодняшний день Git является стандартом индустрии для управления исходным кодом программного обеспечения.

В отличие от традиционных систем, Git не хранит последовательность изменений (дельт), а создаёт полные снимки состояния всех файлов проекта на момент фиксации. Данный подход обеспечивает высокую производительность, целостность данных и гибкость в управлении историей проекта.

2. Зачем использовать Git?

В процессе разработки программного обеспечения часто возникает необходимость возврата к предыдущему рабочему состоянию кода. Это может быть вызвано ошибками, неудачными экспериментами или необходимостью сравнения версий.

Git предоставляет следующие ключевые преимущества:

  • Восстановление предыдущих состояний: Возможность мгновенного возврата к любой зафиксированной версии проекта.
  • Ведение истории изменений: Хранение полной хронологии всех модификаций, включая автора и описание изменений.
  • Безопасность экспериментов: Разработчик может свободно тестировать новые идеи, зная, что всегда может вернуться к стабильной версии.
  • Коллаборация: Поддержка совместной работы нескольких человек над одним проектом без риска потери данных.

Таким образом, Git выступает в роли машины времени для исходного кода, позволяя перемещаться по истории проекта в обоих направлениях.

3. Как работает Git?

Архитектура Git основана на концепции трёх состояний файла и соответствующих им трёх зон хранилища данных.

Три зоны проекта:

  1. Рабочая директория (Working Directory): Локальная копия файлов проекта, с которой непосредственно взаимодействует разработчик.
  2. Индекс (Staging Area): Промежуточная область, в которую помещаются изменения, предназначенные для следующего коммита.
  3. Хранилище Git (Repository / .git): Локальная база данных, содержащая все зафиксированные снимки (коммиты) истории проекта.

Три состояния файла:

  1. Изменён (Modified): Файл в рабочей директории был изменён, но его новая версия ещё не добавлена в индекс.
  2. Подготовлен (Staged): Изменения файла были добавлены в индекс и будут включены в следующий коммит.
  3. Зафиксирован (Committed): Файл сохранён в хранилище Git как часть последнего коммита и является неизменяемой частью истории проекта.

Стандартный рабочий процесс:

Разработчик вносит изменения в файлы → выбирает нужные изменения для сохранения с помощью команды git add → фиксирует их в истории проекта командой git commit.

Команда git status является основным инструментом для отслеживания текущего состояния файлов относительно этих трёх зон.

4. Основные команды Git

Настройка пользователя

Данные команды выполняются единожды для идентификации автора коммитов на данном компьютере.

git config --global user.name "Ваше Имя"
git config --global user.email "ваш_email@example.com"

Инициализация репозитория

Создаёт новый локальный репозиторий Git в текущей директории. Является первым шагом при начале нового проекта.

git init

Проверка состояния

Отображает текущее состояние рабочей директории и индекса: список изменённых, подготовленных и неотслеживаемых файлов.

git status

Добавление файлов в индекс

Перемещает изменения из рабочей директории в индекс, подготавливая их к коммиту.

# Добавить конкретный файл
git add hello.py

# Добавить все изменения в текущей директории
git add .

Создание коммита

Фиксирует снимок всех файлов из индекса в хранилище Git. Обязательно сопровождается описательным сообщением.

git commit -m "Add initial game structure"

Просмотр истории

Выводит список всех коммитов в проекте в хронологическом порядке. Ключ --oneline формирует краткий и удобочитаемый вывод.

git log --oneline

5. Практическое задание

Описание: Создайте консольную программу — текстового бота по имени Вика. Бот должен работать в интерактивном режиме: приветствовать пользователя, принимать его вопросы и выдавать заранее определённые ответы.

Требования к программе:

1. Запуск и основной цикл:

  • При запуске программа выводит на экран:
    Привет! Меня зовут Вика, можешь задать мне любой вопрос.
  • Затем программа выводит приглашение для ввода:
    (0 - выйти) -->
  • Программа ожидает, что пользователь введёт текст.
  • Если пользователь ввёл 0, программа завершает работу и выводит:
    Пока!
  • Если пользователь ввёл что-то другое, программа обрабатывает это сообщение (см. ниже), а затем снова выводит приглашение для ввода:
    (0 - выйти) -->
    — и так по кругу, пока не будет введено 0.

2. Обработка сообщений:

После получения сообщения (не равного 0) программа проверяет его и выводит один из следующих ответов:

  • Если сообщение пустое (пользователь просто нажал Enter), вывести:
    Вы ввели пустое сообщение
  • Если сообщение — привет (маленькими буквами), вывести:
    Привет!
  • Если сообщение — как дела (маленькими буквами), вывести:
    Отлично!
  • Если сообщение — посоветуй фильм (маленькими буквами), вывести:
    Шрек - отличный фильм для семейного просмотра
  • Если сообщение не совпадает ни с одним из вариантов выше, вывести:
    Я не знаю такой команды

Инструкция по выполнению:

  1. Создайте на своём компьютере новую папку для этого задания, например, git_practice.
  2. Откройте эту папку в Visual Studio Code.
  3. Инициализируйте в ней новый Git-репозиторий с помощью команды.
  4. Создайте внутри папки файл с именем bot_vika.py.
  5. Реализуйте программу, следуя требованиям выше.
  6. Важно! После добавления каждого нового правила обработки сообщений (каждого нового условия if или elif) вы должны сделать отдельный коммит:
    • Сначала добавьте файл в индекс.
    • Затем зафиксируйте изменения понятным сообщением.
  7. В результате у вас должно получиться несколько коммитов, каждый из которых представляет собой законченный этап разработки бота.
